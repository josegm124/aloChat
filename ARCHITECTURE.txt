ALO CERTIFAI - MVP ARCHITECTURE

1. Objetivo del MVP

Este repositorio implementa la base de CertifAI como una plataforma de evaluacion de cumplimiento para sistemas de IA de alto riesgo.

Cobertura funcional del MVP:
- Sectores soportados desde el dia 1:
  - Healthcare
  - HR
- Marcos regulatorios base:
  - EU AI Act
  - GDPR
- Idiomas:
  - Espanol (ES)
  - Ingles (EN)
- Entregables al usuario:
  - Reporte web final
  - PDF descargable
  - Notificacion por email con link al reporte

10. Flujo end-to-end completo del MVP

10.1 Web y acceso
Tecnologias:
- Next.js
- Vercel
- ALB
- WAF
- HMAC entre frontend y backend

Paso a paso:
1. El usuario abre la web desplegada en Vercel.
2. La app presenta landing, signup y login.
3. Actualmente signup/login del portal son locales en frontend; no estan conectados todavia a Cognito real.
4. El frontend envia requests al backend usando ALOCHAT_API_BASE_URL y firma HMAC.

10.2 Creacion de assessment
Tecnologias:
- Next.js API routes
- alo-intake-service
- DynamoDB
- Kafka

Paso a paso:
1. El usuario llena el formulario de assessment.
2. El frontend arma CreateAssessmentRequest con metadata del sistema y artifact DOCUMENT_PDF.
3. POST /api/assessment en Next.js proxya al backend.
4. alo-intake-service valida el request.
5. alo-intake-service genera assessmentId, submissionId, traceId e idempotencyKey.
6. alo-intake-service persiste en DynamoDB:
   - alo-assessments
   - alo-submissions
   - alo-idempotency
7. alo-intake-service publica AssessmentIntakeReceivedEvent a alo.assessment.intake.received.

10.3 Profiling regulatorio
Tecnologias:
- alo-profile-service
- Kafka

Paso a paso:
1. alo-profile-service consume alo.assessment.intake.received.
2. Resuelve sector, geografia y categoria de IA.
3. Construye regulatoryProfileId.
4. Determina frameworks aplicables:
   - EU_AI_ACT
   - GDPR
5. Publica ProfileResolvedEvent.

10.4 Analisis documental
Tecnologias:
- alo-document-compliance-service
- PDFBox
- OpenSearch Serverless
- AWS Bedrock
- Kafka

Paso a paso:
1. El frontend envia el PDF a POST /api/v1/document-compliance/analyze.
2. alo-document-compliance-service extrae texto del PDF.
3. Genera preview y evidencia estructurada.
4. Consulta el corpus regulatorio.
5. Genera findings por control y severidad.
6. Devuelve DocumentAnalysisResult.
7. En el flujo async interno tambien publica completion event para el orchestrator.

10.5 Analisis de dataset
Tecnologias:
- alo-dataset-compliance-service
- Kafka

Paso a paso:
1. Si datasetProvided=true, el flujo genera dataset request.
2. alo-dataset-compliance-service analiza csv/xlsx/parquet.
3. Genera findings de governance, calidad y riesgo.
4. Publica DatasetAnalysisCompletedEvent.

10.6 Consolidacion
Tecnologias:
- alo-assessment-orchestrator
- Kafka

Paso a paso:
1. alo-assessment-orchestrator consume document.completed y dataset.completed.
2. Correlaciona por assessmentId.
3. Consolida findings.
4. Calcula:
   - overallScore
   - compliantCount
   - partialCount
   - gapCount
   - criticalGapCount
   - trustSignal
5. Publica ReportGenerationRequestedEvent.

10.7 Generacion del reporte final
Tecnologias:
- alo-report-service
- PDFBox
- S3

Paso a paso:
1. alo-report-service recibe ConsolidatedAssessmentResult.
2. Genera:
   - WebAssessmentReport
   - PdfReportArtifact
3. Persiste en S3:
   - report.json
   - report.html
   - report.pdf
4. Devuelve y expone:
   - reportAccessUrl
   - pdfDownloadUrl

10.8 Email final
Tecnologias:
- alo-notification-service
- Amazon SES

Paso a paso:
1. alo-notification-service genera subject y body bilingue.
2. Usa SES para enviar el correo real.
3. Publica NotificationCompletedEvent.
4. El usuario recibe email con el link al reporte.
